Description

In this stage, you should implement command-line arguments parsing. Parse the -dataType argument and it's following type. The type can be only long, line or word. If the argument is not provided, you should assume that -dataType argument is word.

After that, you must treat all the user inputs as numbers, words or lines depending on command-line argument that was passed to the program.

If the argument is word, the program should separate all the user input into words separated by spaces. You should compare words by length and in case two words have the same length - lexicographically.

If the argument is line, the program should separate all the user input into lines. You should compare two lines by length and in case two lines have the same length - lexicographically, like the words compared. Notice that you should print the longest line on a separate line so you end up with 4 lines being printed, not 2. You can see this in the examples.

You should also add the percentage of occurrence of the found word among all words (words, integers or lines).

By the way, in the case of lines, you should print the longest line on a separate line, but words and numbers should be printed on the same line as percentages. You can see this behavior in the examples.

Examples

Run configuration examples:

java SortingTool -dataType long

java SortingTool -dataType line

java SortingTool -dataType word

Run example:

Input:

1 -2   333 4
42
1                 1

Output example for words:

Total words: 7.
The longest word: 333 (1 time(s), 14%).

Output example for lines:

Total lines: 3.
The longest line:
1                 1
(1 time(s), 33%).

Output example for integers:

Total numbers: 7.
The greatest number: 333 (1 time(s), 14%).
